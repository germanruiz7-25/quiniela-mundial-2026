⚽ Quiniela Mundial 2026 — Guía para montarla en la nube

GitHub Actions (corre solo) + GitHub Pages (enlace público). Desde cero, paso a paso.

Qué vas a lograr: el sistema correrá solo en internet cada 30 minutos, aunque tu PC y tu celular estén apagados. El enlace del tablero se actualizará por su cuenta. Desde el celular solo abres el enlace para ver todo.

Importante sobre quiniela.yml: este archivo NO va suelto. Tiene que ir dentro de una carpeta llamada .github y adentro otra llamada workflows. La ruta final debe ser exactamente: .github/workflows/quiniela.yml. Más abajo te explico cómo crearla en GitHub.

Parte 1 — Crear el repositorio y subir los archivos

Entra a github.com con tu cuenta. Arriba a la derecha, clic en el + y luego New repository.
Ponle un nombre, por ejemplo quiniela-mundial-2026. Marca la opción Private (privado) para que solo tú lo manejes. Clic en Create repository.
En la página del repo nuevo, busca el enlace uploading an existing file (o el botón Add file → Upload files).
Arrastra ahí todos los archivos de la lista de arriba menos quiniela.yml (ese lo subimos aparte). Abajo escribe un mensaje como "primera subida" y clic en Commit changes.
Ahora subimos el quiniela.yml en su carpeta. Clic en Add file → Create new file. En el nombre, escribe exactamente: .github/workflows/quiniela.yml (al escribir las barras /, GitHub crea las carpetas solo).
Abre el archivo quiniela.yml que te di, copia todo su contenido, pégalo en la caja de texto, y clic en Commit changes.

Parte 2 — Guardar tu API key de forma segura

En vez de dejar tu clave escrita en el código, GitHub la guarda como un "secreto" cifrado.

En tu repo, clic en Settings (arriba).
En el menú izquierdo: Secrets and variables → Actions.
Clic en New repository secret.
En Name escribe exactamente: FOOTBALL_API_KEY
En Secret pega tu clave: 3f2be9e89a4540c5bf8ac91e8bd79c59. Clic en Add secret.
Listo. El sistema ya sabe usar tu clave sin que quede visible en el código.

Parte 3 — Permitir que el sistema publique y guarde

Settings → Actions → General. Baja hasta Workflow permissions, elige Read and write permissions y guarda. Esto deja que el sistema guarde los marcadores nuevos.
Settings → Pages. En Source elige Deploy from a branch. En Branch selecciona gh-pages y carpeta / (root), y guarda. (La rama gh-pages aparece sola la primera vez que corre el sistema; si aún no está, vuelve a este paso después de la primera corrida.)

Parte 4 — Primera corrida y tu enlace público

Clic en la pestaña Actions de tu repo. Verás el workflow Actualizar Quiniela.
Clic en él, luego en Run workflow → Run workflow (botón verde). Esto lo corre de una vez sin esperar los 30 minutos.
Espera 1–2 minutos. Cuando aparezca el ✓ verde, ya corrió. Si aparece ✗ rojo, ábrelo para ver el mensaje (casi siempre es un permiso del Paso de la Parte 3).
Tu tablero quedará publicado en: https://TU-USUARIO.github.io/quiniela-mundial-2026/ (cambia TU-USUARIO por tu usuario de GitHub). Ese es el enlace que compartes a todos por WhatsApp.
De aquí en adelante todo es automático. Cada 30 minutos el sistema revisa si terminó algún partido, actualiza los marcadores, recalcula los puntos y refresca ese mismo enlace. No tienes que hacer nada.

Cuando agregues o cambies un participante

Sigues usando tu Excel como siempre: agregas la hoja de la persona, su nombre en la celda C2, sus vaticinios. Luego solo súbelo de nuevo a GitHub:

En tu repo, clic en el archivo 2026_QUINIELA_CONSOLIDADO.xlsm, luego en el ícono de lápiz o en Add file → Upload files para reemplazarlo con tu versión nueva.
Commit changes. Eso es todo: el sistema detecta el Excel nuevo, vuelve a tomar la "foto" de los vaticinios solo, y actualiza el tablero.
La única regla para no romper nada: no muevas de columna el marcador (E y G), el nombre (C2) ni la suma (P8). Agregar hojas de participantes nuevos está perfecto; mover esas celdas, no.

El envío a WhatsApp

Esta parte sigue siendo de un clic manual, por la política de WhatsApp (ningún sistema gratuito puede enviar solo a un número personal). El flujo es sencillo: cuando quieras mandar el avance, abres el enlace del tablero en tu celular, le tomas un screenshot normal, y lo compartes al grupo o al número 50255152112. Como el enlace ya está siempre actualizado, el screenshot siempre muestra los datos más recientes.

Resumen de tu día a día

  • Quieres ver la tabla: abres tu enlace de GitHub Pages desde el celular.
